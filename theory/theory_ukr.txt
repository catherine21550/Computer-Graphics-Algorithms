План розробки калькуляції початкових даних для FPS (Українською)

Цей план розроблено для обчислення початкових параметрів камери та гравця для шутера від першої особи (FPS). 
Він враховує необхідні змінні та їхні взаємозв'язки.

1. Оголошення змінних:

    posX, posY: Координати гравця у світі (float).
    dirX, dirY: Вектор напрямку погляду гравця (float).
    planeX, planeY: Вектор площини камери (float).
    time, oldTime: Змінні для зберігання часу (float або double).

2. Ініціалізація змінних:

    Початкове положення гравця: Задайте posX та posY відповідно до початкового 
	положення гравця у грі. Наприклад:

C++
posX = 5.0f;
posY = 3.0f;

    Початковий напрямок погляду: Задайте dirX та dirY. Важливо, щоб вектор напрямку 
	був нормованим (довжина = 1). Наприклад, для погляду праворуч:

C++
dirX = 1.0f;
dirY = 0.0f;

    Початкова площина камери: planeX та planeY повинні бути перпендикулярними до dirX та dirY. 
	Зазвичай, площина камери коротша за вектор напрямку. Наприклад, для FOV ~66 градусів:

C++
planeX = 0.0f;
planeY = 0.66f;

    Початковий час: Зафіксуйте початковий час для подальшого обчислення FPS та руху:

C++
time = GetTime(); // Функція для отримання поточного часу
oldTime = time;

3. Функція оновлення (game loop):

    Отримання поточного часу:

C++
time = GetTime();

    Обчислення різниці часу (deltaTime):

C++
float deltaTime = time - oldTime;
oldTime = time;

    Оновлення положення гравця: На основі натиснутих клавіш та deltaTime 
	обчислюйте нові posX та posY. Наприклад, для руху вперед:

C++
if (forwardKeyPressed) {
  posX += dirX * moveSpeed * deltaTime;
  posY += dirY * moveSpeed * deltaTime;
}

    Оновлення напрямку погляду та площини камери: При повороті гравця змінюйте dirX, dirY, 
	planeX та planeY відповідно до кута повороту. 
	Важливо зберігати перпендикулярність між вектором напрямку та площиною камери.

    Обчислення та відображення FPS:

C++

float fps = 1.0f / deltaTime;
DrawFPS(fps);

4. Функції:

    GetTime(): Функція для отримання поточного часу.
    DrawFPS(float fps): Функція для відображення FPS на екрані.

5. Зауваження:

    Цей план є базовим. Ви можете додавати інші функції та змінні за необхідності.
    Для реалізації руху та повороту гравця потрібно обробити натискання клавіш.
    Для відображення світу на екрані потрібно використовувати графічну бібліотеку (наприклад, OpenGL, DirectX).

Розширений план розробки калькуляції початкових даних для FPS 
з урахуванням мапи та стін (Українською)

Цей план розширює попередній, додаючи обробку мапи зі стінами та 
перевірку зіткнення, щоб гравець не міг проходити крізь них.

1. Оголошення змінних (доповнено):

    map: Двовимірний масив (вектор векторів або інша структура даних) для зберігання інформації про мапу. 
	Кожен елемент масиву представляє клітинку мапи та містить інформацію про те, чи є там стіна, або інший об'єкт.
    mapWidth, mapHeight: Розміри мапи (цілі числа).
    moveSpeed: Швидкість руху гравця (float).

2. Ініціалізація змінних (доповнено):

    Завантаження мапи: Завантажте дані мапи з файлу або згенеруйте її програмно. 
	Заповніть масив map значеннями, що вказують на стіни та інші об'єкти.
    Розміри мапи: Встановіть mapWidth та mapHeight відповідно до розмірів завантаженої мапи.
    Швидкість руху: Задайте moveSpeed.

3. Функція оновлення (game loop) (доповнено):

    Оновлення положення гравця (з перевіркою зіткнення):

C++
if (forwardKeyPressed) {
    float newPosX = posX + dirX * moveSpeed * deltaTime;
    float newPosY = posY + dirY * moveSpeed * deltaTime;

    // Перевірка, чи нова позиція не знаходиться всередині стіни
    int mapX = (int)newPosX; // Отримуємо індекс клітинки мапи по X
    int mapY = (int)newPosY; // Отримуємо індекс клітинки мапи по Y

    if (mapX >= 0 && mapX < mapWidth 
		&& mapY >= 0 && mapY < mapHeight && map[mapY][mapX] == 0) { // 0 - означає, що тут вільно
        posX = newPosX;
        posY = newPosY;
    }
    //Якщо ж map[mapY][mapX] != 0, то гравець "врізається" в стіну і його координати не змінюються.
}
if (backwardKeyPressed) {
    float newPosX = posX - dirX * moveSpeed * deltaTime;
    float newPosY = posY - dirY * moveSpeed * deltaTime;

    int mapX = (int)newPosX;
    int mapY = (int)newPosY;

    if (mapX >= 0 && mapX < mapWidth && mapY >= 0 && mapY < mapHeight && map[mapY][mapX] == 0) {
        posX = newPosX;
        posY = newPosY;
    }
}

// Аналогічно для руху вліво та вправо.

4. Функції (доповнено):

    LoadMap(const std::string& filename): Функція для завантаження мапи з файлу.
    IsWall(int x, int y): Функція для перевірки, чи є стіна в заданій клітинці мапи. 
	(Можна використовувати безпосередньо звернення до масиву map[y][x]).

5. Структура даних для мапи:

Рекомендується використовувати структуру даних, яка забезпечує ефективний доступ 
до елементів мапи. Наприклад, std::vector<std::vector<int>> або власний клас Map.  
Значення елементів масиву можуть представляти різні типи об'єктів на мапі (стіна, підлога, гравець, тощо).

6. Алгоритм перевірки зіткнення:

    Перетворення координат гравця у координати клітинки мапи.
    Перевірка, чи знаходиться ця клітинка в межах мапи.
    Перевірка, чи є в цій клітинці стіна.

7. Зауваження:

    Цей план враховує просте зіткнення "клітинка-в-клітинку". Для більш точного 
	зіткнення можна використовувати інші алгоритми (наприклад, ray casting або bounding box collision).
    Для відображення мапи на екрані потрібно використовувати графічну бібліотеку.
